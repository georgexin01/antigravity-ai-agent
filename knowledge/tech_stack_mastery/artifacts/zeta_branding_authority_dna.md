## 0. Tier 1: Master Architecture Protocols [ABSOLUTE]

These rules are the highest technical priority and must be followed 100% regardless of other aesthetic choices.

1.  **Web Standards (Vanilla-First)**: All website projects MUST be built as standalone `.html` (Vanilla HTML/CSS/JS) files. **NO VUE** or frameworks for standard websites.
2.  **Multi-Page Architecture**: High-end websites MUST provide depth through distinct pages:
    - **index.html**: Homepage with 8-10+ deeply designed sections.
    - **about.html**: Detailed company history, mission, and registration.
    - **services.html**: Technical breakdown of all core pillars.
    - **portfolio.html**: High-density project gallery.
    - **contact.html**: Mandatory contact form, WhatsApp hooks, and Google Map.
3.  **Hero Slideshow**: The homepage MUST lead with a `100vh` fullscreen **Slideshow** (3+ slides) for maximum visual impact.
4.  **Mobile Format**: Vue.js and modern frameworks are strictly reserved for **Mobile Application** development only.
5.  **Responsive Foundation**: Websites MUST utilize **Bootstrap** (or Bootstrap grid logic) for 100% mobile-responsive layouts.
6.  **Mobile Navigation Logic**: Responsive headers MUST implement a **Popup Menu Slider** (Off-canvas) for mobile interaction to ensure a premium app-like feel.
7.  **Footer Synchronization**: Every page within a website ecosystem MUST share a **100% Identical Footer** (synchronized content, registration, and addresses).

## 3. Major Evolution & Iteration Logic

### I. The Portfolio Pivot (Engagement Flow)

- **Change**: Moving "项目案例库" directly under the Hero slideshow.
- **Logic**: User feedback indicated that "Work Results" are the highest value conversion points. By placing real app screenshots immediately after the hero, we reduce bounce rates and instantly establish authority.
- **Visual Adjustment**: To maintain visual consistency, background patterns (`bg-alt`) were shifted to ensure no segments of the same color touched (Hero -> Dark-Neutral -> Portfolio (Alt-Dark) -> Stats (Dark-Neutral)).

### II. Asset Harvesting Protocol

- **Change**: Transitioning from blank frames to real harvested app screenshots.
- **Logic**: Generic frames look like templates. Real screenshots from `zetacapital.my` (VIPS, BEEKINE, AUTONEST) provide the "Truth Factor".
- **Implementation**: Screenshots were embedded _behind_ a high-resolution iPhone frame (`iphone_cover.png`) using `position: absolute` and `z-index` layering to create depth.

### III. God-Tier Design DNA (Level 2.0.00)

To achieve "God-Tier" status, the following predictive and physiological ux principles were implemented:

- **Core Physics Engine**: Tracks scroll velocity to dynamically update `--velocity-shift`, causing elements to "float" based on user movement speed.
- **Ghost Intent Tracking**: A symbiotic JS engine that monitors cursor proximity, triggering a pre-emptive glow transition (`--ghost-intensity`) before interaction.
- **Liquid Breathing Typography**: Utilizes multi-stop gradients and `@keyframes` to create a living, breathing text effect for core headers.
- **Recursive Layering**: Every main section utilizes multiple pseudo-elements with unique blending modes and structural HUD decorations (Code Rain, Geometric Lines).

## 5. Evolution: From "Neural" to "Engineering Authority"

Today's request to pivot branding from Cyan/Neural to Red/Engineering marks a critical maturation of the site's DNA.

### I. The "Active Trust" Color Theory

- **Cyan (Old)**: Communicated research, observation, and "Future AI" (Passive).
- **Tech Red (Current)**: Communicates engineering kineticism, 24/7 support reliability, and "Software Execution" (Active).
- **Rule**: All future CTA elements must utilize the Red Glow (`var(--color-primary-glow)`) to maintain this active brand posture.

### II. Corporate Infrastructure Mastery

- **Legal Compliance**: The addition of Privacy and Terms pages is not a "filler" task—it is a design requirement for Establishing Authority.
- **Precision Geospatial**: Using specific Google Maps links for the physical office address creates a "Real-World anchor" for a digital agency.

### III. Logo Integrity Protocol (User-Centric Sourcing)

- **Critical Asset Rule**: Branding assets (Logos) are the soul of the project. **NEVER** use AI-generated surrogates as a silent default without exhaustive verification.
- **Deep Sourcing Mandate**: If a logo is missing, the AI must crawl the official domain repeatedly (minimum 5+ passes), inspecting all subdirectories and CSS/JS referenced assets.
- **Intervention Logic**: If deep sourcing fails, the system must **STOP** and present the user with a Choice:
  1. **Upload Assist**: Request the user to provide the official asset.
  2. **Wait Mode**: Halt generation until the brand element is resolved.
  3. **Conditional AI-Gen**: Only proceed with AI-generated branding if the user explicitly chooses the "AI Generation" path from a selection menu.
- **Visual Contrast**: Logos must be sourced or generated to ensure maximum legibility against the theme (e.g., White text for Dark themes), ensuring they stand out distinctly.

## 7. Section Background Mastery & Layering [CRITICAL MANDATE]

To maintain the "Engineering Authority" aesthetic, the following background rules are **MANDATORY**:

### I. Layered Depth Logic

- **Alternating Sections**: Adjacent sections must NEVEr share the same background color.
- **The 10% Rule**: Sections should alternate between `Cyber Black (#020305)` and a slightly lighter `Cyber Neutral (#0c0d12)`. This creates a cognitive "layering" effect that improves page scanability.

### II. Rich Background Utility Protocol

Every landing page must utilize at least 3 of the following "Rich" background types to establish technical density:

- **Liquid Gradient (Breathing)**: Used for high-concept or "ecosystem" sections.
- **Space Stars (Animated)**: Used for portfolio or visionary sections to create "depth-of-field".
- **HUD Geo Art**: Asymmetrical technical overlays (text codes, geometric runes) to break the grid.
- **AI Banner Imagery**: Full-bleed, high-contrast AI-generated tech backgrounds for hero sections.

### III. Mobile Ecosystem Showcase

- **The Frame Rule**: Mobile screenshots must ALWAYS be presented within a high-fidelity device frame (e.g., `iphone_cover.png`) to ground the software in reality.
- **Interactive Focus**: Use "Center Mode" carousels where the active app is framed and side apps are blurred/desaturated, mimicking professional product photography.
