# 📚 LIBRARY INTELLIGENCE (LI) PROTOCOL (V1.0)

The definitive standard for building a self-evolving knowledge ecosystem using **Gemini 3 Flash** (The Hand) and **NotebookLM** (The Mind).

## 🪐 1. THE THREE-LAYER ARCHITECTURE

### Layer 1: Immutable Source (The Pulse)
- **Content**: Raw session logs, mission results, and terminal outputs.
- **Location**: `memory/PULSE.json` and `brain/` logs.
- **Rule**: This data is immutable. It provides the "Ground Truth" for all future learning.

### Layer 2: Synthesized Wiki (The Library)
- **Content**: Concept maps, patterns, architectural blueprints, and cross-project insights.
- **Location**: [Antigravity NotebookLM](https://notebooklm.google.com/notebook/a2ba712a-1285-4f23-bedf-a6b866f32fc3).
- **Rule**: AI MUST use this layer for "Long-Term Memory" to extract patterns without bloating local code context.

### Layer 3: Surgical Schema (The Governance)
- **Content**: Authoritative rules, APEX principles, and Tier-0/1 protocols.
- **Location**: `.gemini/knowledge/`.
- **Rule**: This layer governs AI behavior. It is the "Active Intelligence" that processes Layer 1 and Layer 2.

## 🧬 2. KNOWLEDGE LAYERING (ANTI-SUPPRESSION)

To prevent new information from accidentally deleting or suppressing legacy core principles:

1. **The YAML Stream**: All new knowledge in Layer 3 MUST be appended as "Patches" or "Extensions" rather than direct overwrites of core files when possible.
2. **Versioned Intelligence**: Significant logic shifts must increment the `version` field in the file header.
3. **Legacy Preservation**: AI is FORBIDDEN from deleting Tier-0/1 logic unless it is provably harmful (Conflict Check mandatory).

## 🧪 3. THE DNA SANDBOX (V1.0)

Before new intelligence is promoted to **Tier-1 Core**, it must pass through the **DNA Sandbox**:

1. **Ingest**: New info is gathered from research (NotebookLM/Web).
2. **Staging**: Info is saved to `knowledge/2_governance/dna_candidate.md`.
3. **Consensus**: AI performs a **Generator-Verifier (GV)** loop against existing APEX principles.
4. **Promotion**: Upon user approval ("Go"), the candidate is integrated into Layer 3.
