# Self-Evolving AI Knowledge System V6.0

## 1. CORE EVOLUTION PROTOCOL (HIERARCHICAL)

### 1.1 Observe & Reflect (Real-time)

- **1.1.1 Track Updates**: Discovery of new patterns, tools, or user preferences.
- **1.1.2 Internal Audit**: Ask: "Is this better/faster/more unique than existing knowledge?"

### 1.2 Permanent Recording

- **1.2.1 Knowledge Mapping**:
  - CSS/Design → `website_design_dna.md` or `mobile_app_design_dna.md`
  - Logic/Frameworks → `skills/{tech}/SKILL.md`
  - Workflow Rules → `agent_must_read.md`
- **1.2.2 Deduplication Check**: Rethink before writing to avoid repeating existing rules.

### 1.3 Hierarchical Mandate (MANDATORY)

- All new knowledge MUST be formatted in **numerical steps (1, 1.1, 1.2...)**.

## 2. KNOWLEDGE RETENTION & TOKEN OPTIMIZATION

### 2.1 ADPRS (Advanced Pattern Rating)

- **2.1.1 Axes**: Visual(30%), Structure(25%), UX(20%), Code(15%), Reusability(10%).
- **2.1.2 Mood Signals**:
  - **Positive (+5 to +12)**: "nice", "correct", "perfect", "好", "可以".
  - **Neutral (-3)**: "change abit", "adjust", "改一下".
  - **Negative (-20 to -25)**: "ugly", "dont like", "太丑了", "redo".
- **2.1.3 Thresholds**:
  - **85+ (Starred)**: Prioritize for similar contexts.
  - **50-69 (Warning)**: Trigger A/B testing variations.
  - **Below 30 (Retired)**: Permanently banned and deleted.

### 2.2 Freshness & Decay

- **2.2.1 Time Decay**: Patterns lose -2 points/month if unused.
- **2.2.2 Pruning**: Knowledge not used in 90 days enters Review queue for potential deletion.

## 3. AUTO-DISCOVERY & RESEARCH MANDATE

### 3.1 Daily Skill Scouting (DSS)

- **3.1.1 Search**: 1 session/day via Google (site:github.com, dev.to, Medium).
- **3.1.2 Download**: Max 2 skills/day to `skills/` folder.
- **3.1.3 Safety**: NO executable code, NO obfuscation, NO key harvesting.

### 3.2 Evaluation Cycle

- Test over 2-3 projects. If Rating < 70, delete. If ≥ 70, approve as Permanent.

## 4. MAINTENANCE & HYGIENE

### 4.1 Token Conservation Pass

- Once per month: Merge redundant files, compress verbose descriptions, delete superseding patterns.

### 4.2 File Hierarchy

- `agent_must_read.md` (Master Config) -> `master_strategy.md` (Architecture) -> Specialized KIs.
