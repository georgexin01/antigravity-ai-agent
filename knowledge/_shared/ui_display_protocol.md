# AI Communication UI Display Protocol (V1.2 — GUI Action Layer)

> **Goal**: Transform AI responses into high-fidelity GUI experiences using color-coded Interaction Cards.

## 🟢 VERSION 1: High-Fidelity Markdown (Active)
All replies must utilize these rich-text elements:
*   **Tables**: Mandatory for status and intelligence reports.
*   **Mermaid**: Mandatory for trees, flows, and architectures.
*   **Checkboxes**: Mandatory for task-tracking.

### 🟥 GUI QUESTION BLOCK (Blocking Input)
When the AI needs an answer to move forward:
> [!CAUTION]
> ### 🧠 INPUT REQUIRED: [Topic]
> **Context**: [Summary of the blocker]
> **Options**:
> *   **(A)** [Option 1]
> *   **(B)** [Option 2]
> *   **(C)** [Option 3]
> 
> Status: ⏳ **PENDING FOR USER REPLY**

### 🟩 GUI ACTION CARD (Approval Request)
When the AI is ready to execute a High-Risk or Multi-File plan:
> [!IMPORTANT]
> ### ⚡ EXECUTION READINESS
> **Mission**: [Task Name]
> **Impact**: [Briefly list files/actions]
> **Action**: Please type **"APPROVE"** or **"START"** to execute.

### 🟦 GUI INSIGHT TIP (Predictive Thinking)
When the AI brain is thinking 1-2 steps ahead:
> [!TIP]
> ### 🚀 PREDICTIVE INSIGHT: [Mode Brain]
> I have pre-scaffolded the next logic gate based on your DNA.
> **Logic**: [The +2 Step prediction]

## 🔵 VERSION 2: Cinematic GUI Simulation (Premium)
When a mission reaches 100% completion or a major milestone:
*   **Status Dashboard**: The AI will use `generate_image` to create a "Cinematic Dashboard" (Image) summarizing results.
*   **Live View Intelligence**: The AI will create an HTML/CSS artifact that acts as a "Graphical Interface" for reading complex knowledge logs.
*   **Select/Checkbox Simulation**: Use markdown-based "Interactive Tables" to let the user "Check" or "Select" options visually.

## 📋 Standard UI Components
| Component | Markdown Execution |
| :--- | :--- |
| **Input Simulation** | `[ ] ______________ (Type here)` |
| **Select Simulation** | `( ) Option A | (●) Option B | ( ) Option C` |
| **Checkbox List** | `- [x] Task Done | - [ ] Task Waiting` |
| **Tree Visual** | `Mermaid: graph LR; A-->B; A-->C;` |

## 🔮 VERSION 3: Future-Proofing & Component Monitoring
*Strict mandate for high-fidelity result delivery.*

### 🛠️ PLATFORM COMPONENT SCAN (Step 0.5)
At the start of every session (Step 0.5), the AI must scan for new platform GUI capabilities. 
*   **IF** Real Checkboxes/Radios are supported → **AUTO-UPGRADE** V1.0 Markdown to Real V3.0 GUI.
*   **IF** Real Popups/Modals are supported → **MIGRATE** GitHub Alerts to Real Modals.
*   **ELSE** → Continue with V1.6 Cinematic Markdown.

### 📋 TABLE-FIRST DATA DELIVERY
*   **MANDATORY**: Any result containing more than 3 data points (e.g., progress, logs, schemas, statuses) MUST be formatted as a structured Markdown Table. 
*   **GOAL**: 100% Readability and scan-efficiency for the USER.
