# Premium Corporate Website Design & Development Protocol

_Distilled Intelligence from the JIN HONG M&E "Master Mastery" Project_

## 1. Aesthetic Philosophy: "ThemeForest Excellence"

The goal is to move beyond "flat" or "MVP" designs by embracing **Structural Density** and **Layered Depth**.

### 1.1 Structural Guidelines

- **Density Over Spacing**: Use 4-column grids (`col-lg-3`) for services, projects, and features to give a "full" and professional feel.
- **Micro-Details**: Use CSS patterns like dots (`--pattern-dots`) or lines (`--pattern-lines`) on light backgrounds to prevent visual flatness.
- **Visual Hierarchy**: Use high-contrast "Inverse" styling (Dark Deep Blue card on Light background) to elevate flagship projects or critical conversion points.

## 2. The 5-Tier Background Depth System

Maintain visual rhythm by alternating section backgrounds using the following hierarchy:

| Tier   | Name                 | Use Case                                 | Styling Logic (HSL)                         |
| :----- | :------------------- | :--------------------------------------- | :------------------------------------------ |
| **T1** | **Foundation White** | Core content, long-form text.            | `background: #ffffff;`                      |
| **T2** | **Shallow Blue**     | Subtle branding rhythmic breaks.         | `hsl(220, 90%, 97.2%)`                      |
| **T3** | **Shallow Muted**    | Secondary separation (Services/Process). | `hsl(220, 20%, 94.5%)`                      |
| **T4** | **Premium Deep**     | FAQ, Testimonials, Hero sections.        | `hsl(220, 35%, 15%)`                        |
| **T5** | **Vibrant Gradient** | CTA Banners, Hero Sliders.               | `linear-gradient(135deg, #007bff, #00d2ff)` |

## 3. UI/UX Component Library

### 3.1 Master Button Protocol (8 Variants)

Site-wide consistency is maintained through a structured utility class system:

- `.btn-v3-primary`: The flagship action (Branding color + soft shadow).
- `.btn-v3-vibrant`: Engagement focus (Gradient + glow).
- `.btn-v3-glass`: Hero sections (Backdrop-blur + white border).
- `.btn-v3-cyber`: Sharp industrial edge (Clip-path + secondary border).
- `.btn-v3-soft`: Subtle secondary actions (Low-saturation light BG).
- `.btn-v3-white`: Dark backgrounds (High contrast).
- `.btn-v3-outline-glow`: Modern minimalism (Hover depth).
- `.btn-v3-ghost`: Navigation and subtle footers.

### 3.2 Authority & Trust Modules

- **Floating Trust Ribbon**: Position stats (Years/Workers/Projects) in a floating ribbon _between_ the Hero and the first section. Include reveal animations and subtle floating translations.
- **Visual Qualifications**: Replace text lists with 3D-generated icons (CIDB, ST, SPAN, ISO) in a clean grid.
- **Branding-Linked Footer**: Footer must include Brand Description, Navigation, Live Contact (WA), and Headquarters info + Zeta Capital mandatory credit.

## 4. Engineering "Good Actions"

1.  **Asset Reliability**: Always migrate 3rd party or temporary images to a local `/images/` directory using stable **Relative Paths**.
2.  **Detail Synchronization**: Every interior "Detail" page must perfectly mirror the Homepage's Header, Footer, and Script dependencies.
3.  **Animation Staggering**: Use `.delay-1` through `.delay-4` CSS classes with staggered `animation-delay` for multi-item reveal effects.
4.  **Dark FAQ Pattern**: For dark sections (`Premium Deep`), use transparent FAQ headers with white typography to avoid visual "clutter".
5.  **Sub-Second Conversion**: All header/footer contact points should default to **WhatsApp API** (`wa.me`) for instant lead generation.
